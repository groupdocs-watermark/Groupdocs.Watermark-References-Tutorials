# รายชื่อผู้รับอีเมล Java ด้วย GroupDocs.Watermark

การสกัดที่อยู่ **To**, **CC**, และ **BCC** ทั้งหมดจากไฟล์อีเมลอาจทำให้เหนื่อยล้าเมื่อคุณต้องจัดการกับหลายสิบหรือหลายร้อยข้อความ ในบทแนะนำนี้คุณจะได้เรียนรู้วิธี **list email recipients java** อย่างรวดเร็วและเชื่อถือได้โดยใช้ไลบรารี GroupDocs.Watermark สำหรับ Java เราจะพาคุณผ่านขั้นตอนการตั้งค่า, ตัวอย่างโค้ด, และกรณีการใช้งานจริง เพื่อให้คุณสามารถนำความสามารถนี้ไปผสานในแอปพลิเคชันของคุณได้

## คำตอบสั้น

- **โค้ดนี้ทำอะไร?** เปิดไฟล์อีเมลและพิมพ์ที่อยู่ To, CC, และ BCC ทั้งหมด  
- **ต้องใช้ไลบรารีอะไร?** GroupDocs.Watermark สำหรับ Java (เวอร์ชัน 24.11)  
- **อ่านไฟล์ .msg และ .eml ได้หรือไม่?** ได้ – API รองรับรูปแบบอีเมลทั่วไป  
- **ต้องมีลิขสิทธิ์หรือไม่?** ทดลองใช้ฟรีได้สำหรับการทดสอบ; ต้องมีลิขสิทธิ์เต็มสำหรับการใช้งานจริง  
- **สามารถประมวลผลเป็นชุดได้หรือไม่?** แน่นอน – คุณสามารถวนลูปหลายไฟล์โดยใช้รูปแบบเดียวกัน  

## คำแนะนำเบื้องต้น

คุณรู้สึกเหนื่อยกับการคัดกรองข้อมูลอีเมลด้วยตนเองเพื่อสกัดรายการผู้รับหรือไม่? การทำงานนี้อัตโนมัติจะช่วยประหยัดเวลาและลดข้อผิดพลาด โดยเฉพาะเมื่อจัดการกับปริมาณอีเมลจำนวนมาก คู่มือนี้จะแสดงวิธีใช้ไลบรารี GroupDocs.Watermark สำหรับ Java เพื่อแยกเอกสารอีเมลและ **list email recipients java** อย่างมีประสิทธิภาพ

**สิ่งที่คุณจะได้เรียนรู้**
- การตั้งค่าสภาพแวดล้อมสำหรับการใช้ GroupDocs.Watermark สำหรับ Java  
- การโหลดและเริ่มต้นเอกสารอีเมลด้วย GroupDocs.Watermark API  
- การดึงรายการผู้รับ To, CC, และ BCC จากเอกสารอีเมล  
- การประยุกต์ใช้จริงและข้อควรพิจารณาด้านประสิทธิภาพ  

มาเริ่มต้นด้วยการตรวจสอบข้อกำหนดเบื้องต้นกันเลย

## การตั้งค่า GroupDocs.Watermark สำหรับ Java

เพื่อเริ่มต้น คุณต้องตั้งค่าโปรเจกต์พร้อมกับ dependencies ที่จำเป็น ดำเนินการตามขั้นตอนต่อไปนี้:

**การตั้งค่า Maven**

เพิ่มการกำหนดค่าต่อไปนี้ในไฟล์ `pom.xml` ของคุณเพื่อรวม GroupDocs.Watermark:


**การดาวน์โหลดโดยตรง**

หรือคุณสามารถดาวน์โหลดเวอร์ชันล่าสุดจาก [GroupDocs.Watermark for Java releases](https://releases.groupdocs.com/watermark/java/)

- **ทดลองใช้ฟรี:** เริ่มต้นด้วยการทดลองใช้ฟรีเพื่อสำรวจฟังก์ชันต่าง ๆ

- **ซื้อ:** พิจารณาซื้อลิขสิทธิ์สำหรับการใช้งานในสภาพแวดล้อมการผลิต  

เมื่อการตั้งค่าเสร็จเรียบร้อยแล้ว เราจะเริ่มต้นและเตรียมสภาพแวดล้อมสำหรับการประมวลผลเอกสารอีเมล

## วิธีการ List Email Recipients Java – คู่มือการทำงาน

ส่วนนี้จะแบ่งคุณลักษณะแต่ละอย่างเป็นขั้นตอนย่อย เพื่อให้คุณสามารถนำการแยกอีเมลไปใช้กับ GroupDocs.Watermark ได้อย่างมีประสิทธิภาพ

### โหลดและเริ่มต้นเอกสารอีเมล

**ภาพรวม**  
การโหลดเอกสารอีเมลเป็นขั้นตอนแรกของกระบวนการ นี้รวมถึงการสร้างอ็อบเจกต์ `Watermarker` ซึ่งทำหน้าที่เป็นประตูสู่การทำงานกับไฟล์อีเมล

**ขั้นตอนการทำงาน**

1. **นำเข้าคลาสที่จำเป็น**  
   ```java
   import com.groupdocs.watermark.Watermarker;
   import com.groupdocs.watermark.options.EmailLoadOptions;
   ```
2. **กำหนดเส้นทางไฟล์อีเมลและตัวเลือกการโหลด**  
   ระบุเส้นทางไปยังเอกสารอีเมลของคุณ แทนที่ `"YOUR_DOCUMENT_DIRECTORY/email.msg"` ด้วยเส้นทางจริงของไฟล์  
   ```java
   String emailFilePath = "YOUR_DOCUMENT_DIRECTORY/email.msg";
   EmailLoadOptions loadOptions = new EmailLoadOptions();
   Watermarker watermarker = new Watermarker(emailFilePath, loadOptions);
   ```
3. **การจัดการทรัพยากร**  
   อย่าลืมปิดอินสแตนซ์ `Watermarker` หลังการใช้งานเพื่อปล่อยทรัพยากรระบบ  
   ```java
   watermarker.close();
   ```

### รายชื่อผู้รับโดยตรง (To) ทั้งหมดของอีเมล

**ภาพรวม**  
การดึงผู้รับโดยตรง (To) ทำได้ง่ายเมื่อคุณได้เริ่มต้นเอกสารอีเมลแล้ว

**ขั้นตอนการทำงาน**

1. **ดึงเนื้อหาอีเมล**  
   ตรวจสอบให้แน่ใจว่าอ็อบเจกต์ `watermarker` ถูกเริ่มต้นแล้วตามที่แสดงในส่วนก่อนหน้า  
   ```java
   import com.groupdocs.watermark.contents.EmailContent;
   
   EmailContent content = watermarker.getContent(EmailContent.class);
   ```
2. **วนลูปและแสดงผู้รับ**  
   ทำการวนลูปผ่านรายการผู้รับโดยตรงและพิมพ์ที่อยู่อีเมลแต่ละรายการ  
   ```java
   for (EmailAddress address : content.getTo()) {
       System.out.println("Direct Recipient: " + address.getEmailAddress());
   }
   ```

### รายชื่อผู้รับ CC ทั้งหมดของอีเมล

**ภาพรวม**  
การแสดงผู้รับ CC ทำตามกระบวนการคล้ายกับการแสดงผู้รับโดยตรง ช่วยให้คุณเข้าถึงที่อยู่อีเมลเพิ่มเติมในฟิลด์ CC

**ขั้นตอนการทำงาน**

1. **ดึงและวนลูป**  
   ใช้วัตถุ `EmailContent` จากขั้นตอนก่อนหน้า:  
   ```java
   for (EmailAddress address : content.getCc()) {
       System.out.println("CC Recipient: " + address.getEmailAddress());
   }
   ```

### รายชื่อผู้รับ BCC ทั้งหมดของอีเมล

**ภาพรวม**  
แม้ว่าผู้รับ BCC จะไม่ปรากฏในส่วนหัวของอีเมล คุณยังสามารถดึงข้อมูลเหล่านี้ได้ด้วย GroupDocs.Watermark

**ขั้นตอนการทำงาน**

1. **เข้าถึงและแสดงที่อยู่ BCC**  
   ```java
   for (EmailAddress address : content.getBcc()) {
       System.out.println("BCC Recipient: " + address.getEmailAddress());
   }
   ```

## การประยุกต์ใช้จริง

คุณลักษณะเหล่านี้สามารถผสานเข้ากับระบบต่าง ๆ ได้ เช่น:

- **ระบบจัดการอีเมล:** ทำให้การจัดหมวดหมู่และประมวลผลอีเมลอัตโนมัติตามรายการผู้รับเป็นเรื่องง่าย  
- **เครื่องมือวิเคราะห์ข้อมูล:** สกัดข้อมูลผู้รับเพื่อทำการวิเคราะห์และระบุรูปแบบการสื่อสารภายในองค์กร  
- **ซอฟต์แวร์ด้านความปลอดภัย:** ตรวจสอบการจราจรของอีเมลเพื่อค้นหาการแชร์ข้อมูลโดยไม่ได้รับอนุญาตหรือการรั่วไหล  

## ข้อควรพิจารณาด้านประสิทธิภาพ

เมื่อจัดการกับอีเมลจำนวนมาก ให้คำนึงถึงเคล็ดลับต่อไปนี้:

- **เพิ่มประสิทธิภาพการใช้ทรัพยากร:** ปิดอ็อบเจกต์ `Watermarker` ทันทีหลังการใช้งาน  
- **การจัดการหน่วยความจำ:** ระวังการทำงานของ Garbage Collector ของ Java และการใช้หน่วยความจำเมื่อประมวลผลหลายไฟล์พร้อมกัน  
- **การประมวลผลเป็นชุด:** จัดการอีเมลเป็นชุดเพื่อช่วยลดภาระบนระบบ  

## คำถามที่พบบ่อย

**Q: จะจัดการข้อผิดพลาดระหว่างการแยกอีเมลอย่างไร?**  
A: ตรวจสอบให้แน่ใจว่าเส้นทางไฟล์ถูกต้อง ไฟล์เป็นรูปแบบที่รองรับ และห่อโค้ดของคุณด้วยบล็อก `try‑catch` เพื่อจับ `IOException` หรือ `GroupDocsException`

**Q: สามารถใช้ไลบรารีนี้กับรูปแบบอีเมลอื่นเช่น .eml ได้หรือไม่?**  
A: ได้, GroupDocs.Watermark รองรับรูปแบบอีเมลหลายประเภท ตรวจสอบเอกสารสำหรับตัวเลือกการโหลดที่เฉพาะเจาะจงกับรูปแบบนั้น

**Q: จุดบกพร่องทั่วไปเมื่อทำรายการผู้รับคืออะไร?**  
A: เส้นทางไฟล์ไม่ถูกต้อง, ประเภทไฟล์ที่ไม่รองรับ, หรือการลืมปิดอินสแตนซ์ `Watermarker` ซึ่งอาจทำให้เกิดการรั่วไหลของทรัพยากร

**Q: จะเพิ่มประสิทธิภาพเมื่อแยกอีเมลจำนวนมากอย่างไร?**  
A: ประมวลผลไฟล์แบบขนานด้วย `ExecutorService` ของ Java แต่ต้องเฝ้าติดตามการใช้ CPU และหน่วยความจำเพื่อหลีกเลี่ยงการโอเวอร์โหลด

**Q: จะหาความช่วยเหลือเมื่อเจอปัญหาได้จากที่ไหน?**  
A: เยี่ยมชม [GroupDocs Free Support Forum](https://forum.groupdocs.com/c/watermark/10) เพื่อรับการช่วยเหลือจากชุมชนและทีมสนับสนุนอย่างเป็นทางการ
