# Προσθήκη Υδατογραφήματος Java: Φόρτωση & Αποθήκευση Εγγράφων με GroupDocs.Watermark

Σε αυτόν τον οδηγό θα ανακαλύψετε **how to add watermark java** σε ένα ευρύ φάσμα τύπων εγγράφων, να φορτώσετε αυτά τα έγγραφα από δίσκο, ροές ή άλλες πηγές, και τελικά να αποθηκεύσετε τα αρχεία με υδατογράφημα. Είτε δημιουργείτε μια υπηρεσία επεξεργασίας παρτίδας είτε έναν ανεβάστη σελίδας, τα παρακάτω βήματα σας παρέχουν μια σαφή, ολοκληρωμένη ροή εργασίας χρησιμοποιώντας το GroupDocs.Watermark για Java.

## Γρήγορες Απαντήσεις
- **Τι κάνει το “add watermark java”;** Ενσωματώνει κείμενα ή εικόνες ως υδατογραφήματα σε υποστηριζόμενες μορφές εγγράφων μέσω του GroupDocs.Watermark Java API.  
- **Μπορώ να φορτώσω ένα έγγραφο από ροή;** Ναι – το API δέχεται αντικείμενα `InputStream`, καθιστώντας εύκολο την εργασία με αρχεία που αποθηκεύονται στη μνήμη ή λαμβάνονται από δίκτυο.  
- **Πώς διαχειρίζονται τα αρχεία με προστασία κωδικού;** Περνάτε τον κωδικό κατά τη δημιουργία ενός αντικειμένου `Watermark`; η βιβλιοθήκη θα αποκρυπτογραφήσει, θα εφαρμόσει τα υδατογραφήματα και θα κρυπτογραφήσει ξανά το αρχείο.  
- **Ποιες μορφές υποστηρίζονται;** PDF, DOC/DOCX, PPT/PPTX, XLS/XLSX, εικόνες (PNG, JPEG, BMP) και πολλά άλλα.  
- **Χρειάζομαι άδεια για παραγωγή;** Απαιτείται εμπορική άδεια για χρήση σε παραγωγή· διατίθεται δωρεάν δοκιμή για αξιολόγηση.

## Τι είναι το “add watermark java”;
Το “add watermark java” αναφέρεται στη διαδικασία προγραμματιστικής εισαγωγής ορατών ή αόρατων υδατογραφημάτων σε έγγραφα χρησιμοποιώντας τη βιβλιοθήκη GroupDocs.Watermark γραμμένη σε Java. Αυτή η τεχνική βοηθά στην προστασία της πνευματικής ιδιοκτησίας, στην επωνυμία των εγγράφων ή στην τήρηση κανονιστικών απαιτήσεων.

## Γιατί να χρησιμοποιήσετε το GroupDocs.Watermark για Java;
- **Ευρεία υποστήριξη μορφών** – ένα API λειτουργεί σε PDF, αρχεία Office και εικόνες.  
- **Φιλικό προς τις ροές** – φόρτωση από `FileInputStream`, `ByteArrayInputStream` ή οποιαδήποτε προσαρμοσμένη ροή χωρίς να αγγίζετε το σύστημα αρχείων.  
- **Διαχείριση κωδικού** – ενσωματωμένη υποστήριξη για άνοιγμα και αποθήκευση κρυπτογραφημένων εγγράφων.  
- **Υψηλή απόδοση** – βελτιστοποιημένο για μεγάλα αρχεία και λειτουργίες παρτίδας.  
- **Απλό API** – σαφείς, ρευστές μέθοδοι που διατηρούν τον κώδικά σας αναγνώσιμο και συντηρήσιμο.

## Προαπαιτούμενα
- Εγκατεστημένο Java 8 ή νεότερο.  
- Προσθήκη της βιβλιοθήκης GroupDocs.Watermark for Java στο έργο σας (Maven/Gradle).  
- Έγκυρη άδεια GroupDocs.Watermark για παραγωγή (προαιρετική για δοκιμές).

## Οδηγός Βήμα‑βήμα

### Βήμα 1: Ρύθμιση του έργου
Προσθέστε την εξάρτηση GroupDocs.Watermark στο `pom.xml` (ή στο αρχείο Gradle) και συμπεριλάβετε το κλειδί άδειας στην κωδικοποίηση εκκίνησης.

### Βήμα 2: Φόρτωση εγγράφου από δίσκο ή ροή
Χρησιμοποιήστε την κλάση `Watermark` για να ανοίξετε ένα αρχείο απευθείας από διαδρομή, ή περάστε ένα `InputStream` όταν το έγγραφο βρίσκεται στη μνήμη ή σε απομακρυσμένη θέση.

### Βήμα 3: Εφαρμογή κειμενικού ή εικόνας υδατογραφήματος
Δημιουργήστε ένα αντικείμενο `TextWatermark` ή `ImageWatermark`, ρυθμίστε την εμφάνισή του (χρώμα, διαφάνεια, περιστροφή) και προσθέστε το στο φορτωμένο έγγραφο.

### Βήμα 4: Αποθήκευση του εγγράφου με υδατογράφημα
Επιλέξτε τη μορφή εξόδου (ίδια με την πηγή ή διαφορετική) και γράψτε το αποτέλεσμα σε αρχείο, ροή ή πίνακα byte.

### Βήμα 5: Διαχείριση αρχείων με προστασία κωδικού (προαιρετικό)
Κατά το άνοιγμα ενός προστατευμένου εγγράφου, δώστε τον κωδικό στις επιλογές φόρτωσης. Μετά το υδατογράφημα, η βιβλιοθήκη εφαρμόζει ξανά την κρυπτογράφηση αυτόματα.

## Συχνά Προβλήματα & Λύσεις
- **Το έγγραφο δεν φορτώνει από ροή** – βεβαιωθείτε ότι η ροή έχει επαναφερθεί (`stream.reset()`) πριν τη περάσετε στο API.  
- **Το υδατογράφημα δεν είναι ορατό** – ελέγξτε ότι η διαφάνεια και η αντίθεση χρώματος είναι κατάλληλες για τη μορφή-στόχο.  
- **Η αποθήκευση αποτυγχάνει για μεγάλα PDF** – αυξήστε το μέγεθος heap του JVM ή χρησιμοποιήστε τη μέθοδο `Document.optimizeResources()` για μείωση της κατανάλωσης μνήμης.
