# Создание текстового водяного знака на диаграммах с GroupDocs.Watermark Java

Защита ваших диаграмм от несанкционированного использования является обязательной в современных совместных средах. В этом руководстве вы **создадите текстовый водяной знак** на диаграммах в стиле Visio, используя **GroupDocs.Watermark for Java**. Мы пройдем весь процесс от настройки Maven до сохранения файла с водяным знаком, чтобы вы могли уверенно добавлять брендинг или элементы безопасности на каждую страницу диаграммы.

## Быстрые ответы
- **Какую библиотеку мне нужно?** GroupDocs.Watermark for Java (Maven артефакт `groupdocs-watermark`).
- **Какие типы файлов поддерживаются?** Visio (`.vsdx`, `.vsd`), а также многие другие форматы диаграмм.
- **Нужна ли лицензия?** Временная пробная лицензия подходит для разработки; полная лицензия требуется для продакшн.
- **Можно ли настроить шрифт, цвет и поворот?** Да — класс `TextWatermark` позволяет задать все эти свойства.
- **Сколько времени занимает процесс?** Добавление текстового водяного знака к типичной диаграмме занимает менее секунды на современном оборудовании.

## Что такое операция «создание текстового водяного знака»?
Создание текстового водяного знака означает программное наложение полупрозрачного текста на страницу документа. Водяной знак может быть размещён на переднем или заднем плане, вращён, окрашен и стилизован в соответствии с требованиями брендинга или безопасности.

## Почему использовать GroupDocs.Watermark for Java?
- **Широкая поддержка форматов** — работает с Visio, PDF, Word, Excel и другими.
- **Тонкая настройка** — выбирайте размещение, непрозрачность, поворот и рендеринг заднего/переднего плана.
- **Лёгкая интеграция** — настройка на основе Maven и простые вызовы API позволяют поддерживать чистый код.
- **Оптимизированная производительность** — ресурсы освобождаются автоматически при закрытии `Watermarker`.

## Предварительные требования
- Java Development Kit (JDK) 8 или выше.
- IDE, например IntelliJ IDEA или Eclipse.
- Maven для управления зависимостями.

Добавьте репозиторий и зависимость в ваш `pom.xml`:


## Пошаговая реализация

### Шаг 1: Загрузка файла диаграммы
Используйте `DiagramLoadOptions` для открытия файла Visio (или любого поддерживаемого формата диаграмм).

```java
String inputFilePath = "YOUR_DOCUMENT_DIRECTORY/diagram.vsdx";
Watermarker watermarker = new Watermarker(inputFilePath, new DiagramLoadOptions());
```

### Шаг 2: Создание текстового водяного знака
Настройте текст водяного знака, шрифт, цвет, флаг заднего плана и угол поворота.

```java
TextWatermark textWatermark = new TextWatermark("Test watermark", new Font("Arial", 36));
textWatermark.setColor(Color.getBlue());
textWatermark.setBackground(false);
textWatermark.setRotationAngle(-45);
```

### Шаг 3: Определение размещения для диаграммы
Выберите, будет ли водяной знак отображаться на заднем или переднем плане каждой страницы диаграммы.

```java
DiagramShapeWatermarkOptions options = new DiagramShapeWatermarkOptions();
options.setPlacement(DiagramWatermarkPlacementType.Background);
watermarker.add(textWatermark, options);
```

### Шаг 4: Сохранение диаграммы с водяным знаком
Запишите результат в новый файл и освободите ресурсы.

```java
String outputFilePath = "YOUR_OUTPUT_DIRECTORY/watermarked_diagram.vsdx";
watermarker.save(outputFilePath);
watermarker.close();
```

| Проблема | Решение |
|---------|----------|
| **Файл не найден** | Проверьте абсолютные/относительные пути и убедитесь, что файл доступен для чтения процессом Java. |
| **Лицензия не применена** | Убедитесь, что путь к файлу лицензии правильный и файл не повреждён. |
| **Водяной знак не виден** | Проверьте `setBackground(false)` и угол поворота; попробуйте другой цвет или непрозрачность. |
| **Неподдерживаемая версия диаграммы** | Используйте последнюю версию GroupDocs.Watermark (24.11), которая добавляет поддержку новых форматов Visio. |

## Раздел FAQ
1. **Можно ли использовать GroupDocs.Watermark для других форматов файлов?**  
   Да, он поддерживает PDF, документы Word, листы Excel, презентации PowerPoint и многое другое.
2. **Есть ли ограничение на количество добавляемых водяных знаков?**  
   Жёсткого ограничения нет, но добавление большого количества сложных водяных знаков может повлиять на производительность.
3. **Как удалить водяной знак из диаграммы?**  
   GroupDocs.Watermark предоставляет API для обнаружения и удаления, которые можно вызывать аналогично процессу добавления.
4. **Можно ли добавить текстовые водяные знаки на все страницы или только выбранные?**  
   Вы можете настроить `DiagramShapeWatermarkOptions` для каждой страницы или применить фильтры перед вызовом `add`.
5. **Что делать, если водяной знак не отображается как ожидалось?**  
   Тщательно проверьте настройки размещения, контрастность цвета и убедитесь, что диаграмма не уплощается после сохранения.

## Часто задаваемые вопросы

**В: Работает ли библиотека с файлами Visio 2003 (`.vsd`)?**  
О: Да — `DiagramLoadOptions` поддерживает как форматы `.vsdx`, так и устаревшие `.vsd`.

**В: Можно ли изменить непрозрачность текстового водяного знака?**  
О: Конечно. Используйте `textWatermark.setOpacity(0.5);`, чтобы установить 50 % прозрачности.

**В: Можно ли добавить разные водяные знаки на разные страницы диаграммы?**  
О: Да. Проходите по `watermarker.getPages()` и применяйте разные экземпляры `TextWatermark` с индивидуальными параметрами для каждой страницы.

**В: Есть ли ограничения лицензирования для коммерческого использования?**  
О: Для продакшн‑развёртываний требуется полная коммерческая лицензия; пробная лицензия предназначена только для оценки.

**В: Как выполнить пакетную обработку нескольких диаграмм за один запуск?**  
О: Оберните вышеописанные шаги в цикл, который загружает каждый файл, применяет водяной знак, сохраняет и закрывает `Watermarker` перед переходом к следующему файлу.
