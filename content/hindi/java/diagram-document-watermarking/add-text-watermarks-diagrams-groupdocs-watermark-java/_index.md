# डायग्राम पर टेक्स्ट वॉटरमार्क बनाएं GroupDocs.Watermark Java के साथ

अपने डायग्राम को अनधिकृत पुन: उपयोग से बचाना आज के सहयोगी वातावरण में अनिवार्य है। इस ट्यूटोरियल में आप **टेक्स्ट वॉटरमार्क** Visio‑स्टाइल डायग्राम पर **GroupDocs.Watermark for Java** का उपयोग करके बनाएंगे। हम Maven सेटअप से लेकर वॉटरमार्केड फ़ाइल को सेव करने तक सभी चरणों को कवर करेंगे, ताकि आप प्रत्येक डायग्राम पेज में भरोसेमंद रूप से ब्रांडिंग या सुरक्षा चिह्न जोड़ सकें।

## त्वरित उत्तर
- **मुझे कौनसी लाइब्रेरी चाहिए?** GroupDocs.Watermark for Java (Maven artifact `groupdocs-watermark`).
- **कौनसे फ़ाइल प्रकार समर्थित हैं?** Visio (`.vsdx`, `.vsd`), साथ ही कई अन्य डायग्राम फ़ॉर्मेट।
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए एक अस्थायी ट्रायल लाइसेंस काम करता है; प्रोडक्शन के लिए पूर्ण लाइसेंस आवश्यक है।
- **क्या मैं फ़ॉन्ट, रंग, और घुमाव को कस्टमाइज़ कर सकता हूँ?** हाँ – `TextWatermark` क्लास आपको इन सभी प्रॉपर्टीज़ को सेट करने देता है।
- **प्रक्रिया में कितना समय लगता है?** सामान्य डायग्राम में टेक्स्ट वॉटरमार्क जोड़ने में आधुनिक हार्डवेयर पर एक सेकंड से कम समय लगता है।

## “टेक्स्ट वॉटरमार्क बनाना” ऑपरेशन क्या है?
टेक्स्ट वॉटरमार्क बनाना मतलब प्रोग्रामेटिक रूप से दस्तावेज़ पेज पर अर्ध‑पारदर्शी टेक्स्ट ओवरले करना है। वॉटरमार्क को अग्रभूमि या पृष्ठभूमि में रखा जा सकता है, घुमाया, रंगा और ब्रांडिंग या सुरक्षा आवश्यकताओं के अनुसार स्टाइल किया जा सकता है।

## GroupDocs.Watermark for Java का उपयोग क्यों करें?
- **व्यापक फ़ॉर्मेट समर्थन** – Visio, PDF, Word, Excel, और अधिक के साथ काम करता है।
- **सूक्ष्म नियंत्रण** – प्लेसमेंट, अपारदर्शिता, घुमाव, और पृष्ठभूमि/अग्रभूमि रेंडरिंग चुनें।
- **आसान इंटीग्रेशन** – Maven‑आधारित सेटअप और सरल API कॉल्स आपके कोड को साफ़ रखते हैं।
- **प्रदर्शन‑अनुकूलित** – जब आप `Watermarker` को बंद करते हैं तो संसाधन स्वचालित रूप से रिलीज़ हो जाते हैं।

## आवश्यकताएँ
- Java Development Kit (JDK) 8 या उससे ऊपर।
- IntelliJ IDEA या Eclipse जैसे IDE।
- निर्भरता प्रबंधन के लिए Maven।

## GroupDocs.Watermark for Java सेटअप करना
`pom.xml` में रिपॉजिटरी और डिपेंडेंसी जोड़ें:

```xml
<repositories>
   <repository>
      <id>repository.groupdocs.com</id>
      <name>GroupDocs Repository</name>
      <url>https://releases.groupdocs.com/watermark/java/</url>
   </repository>
</repositories>

<dependencies>
   <dependency>
      <groupId>com.groupdocs</groupId>
      <artifactId>groupdocs-watermark</artifactId>
      <version>24.11</version>
   </dependency>
</dependencies>
```

यदि आप मैन्युअल डाउनलोड पसंद करते हैं, तो बाइनरीज़ को [GroupDocs.Watermark for Java releases](https://releases.groupdocs.com/watermark/java/) से प्राप्त करें और उन्हें अपने प्रोजेक्ट के क्लासपाथ में जोड़ें।

### लाइसेंस प्राप्ति
एक मुफ्त ट्रायल लाइसेंस के साथ शुरू करें [GroupDocs.Trial Licensing](https://purchase.groupdocs.com/temporary-license/) से। किसी भी वॉटरमार्क ऑपरेशन से पहले लाइसेंस फ़ाइल लोड करें:

```java
License license = new License();
license.setLicense("path/to/license/file");
```

## चरण‑दर‑चरण कार्यान्वयन

### चरण 1: डायग्राम फ़ाइल लोड करें
`DiagramLoadOptions` का उपयोग करके Visio फ़ाइल (या कोई भी समर्थित डायग्राम फ़ॉर्मेट) खोलें।

```java
String inputFilePath = "YOUR_DOCUMENT_DIRECTORY/diagram.vsdx";
Watermarker watermarker = new Watermarker(inputFilePath, new DiagramLoadOptions());
```

### चरण 2: टेक्स्ट वॉटरमार्क बनाएं
वॉटरमार्क टेक्स्ट, फ़ॉन्ट, रंग, बैकग्राउंड फ़्लैग, और घुमाव कोण को कॉन्फ़िगर करें।

```java
TextWatermark textWatermark = new TextWatermark("Test watermark", new Font("Arial", 36));
textWatermark.setColor(Color.getBlue());
textWatermark.setBackground(false);
textWatermark.setRotationAngle(-45);
```

### चरण 3: डायग्राम के लिए प्लेसमेंट निर्धारित करें
निर्धारित करें कि वॉटरमार्क प्रत्येक डायग्राम पेज की पृष्ठभूमि में या अग्रभूमि में दिखेगा।

```java
DiagramShapeWatermarkOptions options = new DiagramShapeWatermarkOptions();
options.setPlacement(DiagramWatermarkPlacementType.Background);
watermarker.add(textWatermark, options);
```

### चरण 4: वॉटरमार्केड डायग्राम को सेव करें
परिणाम को नई फ़ाइल में लिखें और संसाधनों को रिलीज़ करें।

```java
String outputFilePath = "YOUR_OUTPUT_DIRECTORY/watermarked_diagram.vsdx";
watermarker.save(outputFilePath);
watermarker.close();
```

## सामान्य समस्याएँ और समाधान

| समस्या | समाधान |
|---------|----------|
| **फ़ाइल नहीं मिली** | एब्सोल्यूट/रिलेटिव पाथ्स की जाँच करें और सुनिश्चित करें कि फ़ाइल Java प्रोसेस द्वारा पढ़ी जा सकती है। |
| **लाइसेंस लागू नहीं हुआ** | लाइसेंस फ़ाइल पाथ सही है और फ़ाइल भ्रष्ट नहीं है, यह पुष्टि करें। |
| **वॉटरमार्क दिखाई नहीं दे रहा** | `setBackground(false)` और घुमाव कोण की जाँच करें; अलग रंग या अपारदर्शिता आज़माएँ। |
| **असमर्थित डायग्राम संस्करण** | नवीनतम GroupDocs.Watermark संस्करण (24.11) का उपयोग करें जो नए Visio फ़ॉर्मेट्स के लिए समर्थन जोड़ता है। |

## व्यावहारिक उपयोग केस
1. **डॉक्यूमेंट सुरक्षा** – प्रतिस्पर्धियों को स्वामित्व वाले फ्लोचार्ट को पुनः उपयोग करने से रोकें।
2. **ब्रांड स्थिरता** – सभी एक्सपोर्टेड डायग्राम में कंपनी का नाम या लोगो स्वचालित रूप से एम्बेड करें।
3. **सहयोग ट्रैकिंग** – प्रत्येक डायग्राम को किसने संपादित किया, यह पहचानने के लिए उपयोगकर्ता के इनिशियल्स को वॉटरमार्क के रूप में जोड़ें।

## प्रदर्शन टिप्स
- जब आप समाप्त कर लें तो `Watermarker` को तुरंत बंद करें ताकि नेटिव संसाधन मुक्त हो सकें।
- बैच प्रोसेसिंग के लिए, संभव हो तो एक ही `Watermarker` इंस्टेंस को पुन: उपयोग करें।
- वॉटरमार्क टेक्स्ट को संक्षिप्त रखें; बड़े फ़ॉन्ट्स रेंडरिंग समय बढ़ाते हैं।

## निष्कर्ष
अब आप जानते हैं कि GroupDocs.Watermark for Java का उपयोग करके Visio डायग्राम पर **टेक्स्ट वॉटरमार्क** कैसे बनाएं। यह तरीका आपको रूप, प्लेसमेंट और स्टाइलिंग पर पूर्ण नियंत्रण देता है, जिससे आप अपने विज़ुअल एसेट्स को प्रभावी रूप से सुरक्षित और ब्रांड कर सकते हैं।

### अगले कदम
- लोगो ब्रांडिंग के लिए इमेज वॉटरमार्क (`ImageWatermark`) के साथ प्रयोग करें।
- `watermarker.getPages()` पर इटररेट करके और शर्तानुसार विकल्प लागू करके चयनात्मक पेज वॉटरमार्किंग का अन्वेषण करें।
- इस लॉजिक को अपने CI/CD पाइपलाइन में इंटीग्रेट करें ताकि प्रकाशित करने से पहले डायग्राम स्वचालित रूप से सुरक्षित हो जाएँ।

## अक्सर पूछे जाने वाले प्रश्न
1. **क्या मैं GroupDocs.Watermark को अन्य फ़ाइल फ़ॉर्मेट्स के लिए उपयोग कर सकता हूँ?**  
   हाँ, यह PDFs, Word दस्तावेज़, Excel शीट्स, PowerPoint डेक्स, और कई अन्य को सपोर्ट करता है।

2. **क्या मैं जोड़ सकने वाले वॉटरमार्क की संख्या पर कोई सीमा है?**  
   कोई कठोर सीमा नहीं है, लेकिन बहुत सारे जटिल वॉटरमार्क जोड़ने से प्रदर्शन पर असर पड़ सकता है।

3. **मैं डायग्राम से वॉटरमार्क कैसे हटाऊँ?**  
   GroupDocs.Watermark डिटेक्शन और रिमूवल API प्रदान करता है जिसे आप जोड़ने के फ्लो की तरह कॉल कर सकते हैं।

4. **क्या टेक्स्ट वॉटरमार्क सभी पेजों पर या केवल चयनित पेजों पर जोड़े जा सकते हैं?**  
   आप प्रत्येक पेज के लिए `DiagramShapeWatermarkOptions` कॉन्फ़िगर कर सकते हैं या `add` कॉल करने से पहले फ़िल्टर लागू कर सकते हैं।

5. **यदि वॉटरमार्क अपेक्षित रूप से नहीं दिख रहा है तो क्या करें?**  
   प्लेसमेंट सेटिंग्स, रंग कंट्रास्ट की दोबारा जाँच करें, और सुनिश्चित करें कि सहेजने के बाद डायग्राम फ्लैट नहीं हो रहा है।

## अक्सर पूछे जाने वाले प्रश्न
**प्रश्न: क्या लाइब्रेरी Visio 2003 (`.vsd`) फ़ाइलों के साथ काम करती है?**  
**उत्तर:** हाँ – `DiagramLoadOptions` दोनों `.vsdx` और लेगेसी `.vsd` फ़ॉर्मेट को सपोर्ट करता है।

**प्रश्न: क्या मैं टेक्स्ट वॉटरमार्क की अपारदर्शिता बदल सकता हूँ?**  
**उत्तर:** बिल्कुल। `textWatermark.setOpacity(0.5);` का उपयोग करके 50 % ट्रांसपेरेंसी सेट करें।

**प्रश्न: क्या विभिन्न डायग्राम पेजों पर अलग-अलग वॉटरमार्क जोड़ना संभव है?**  
**उत्तर:** हाँ। `watermarker.getPages()` पर इटररेट करें और प्रत्येक पेज के लिए कस्टम विकल्पों के साथ अलग `TextWatermark` इंस्टेंस लागू करें।

**प्रश्न: व्यावसायिक उपयोग के लिए कोई लाइसेंस प्रतिबंध हैं?**  
**उत्तर:** प्रोडक्शन डिप्लॉयमेंट के लिए पूर्ण व्यावसायिक लाइसेंस आवश्यक है; ट्रायल लाइसेंस केवल मूल्यांकन के लिए है।

**प्रश्न: मैं एक रन में कई डायग्राम को बैच‑प्रोसेस कैसे कर सकता हूँ?**  
**उत्तर:** ऊपर के चरणों को एक लूप में रखें जो प्रत्येक फ़ाइल को लोड करे, वॉटरमार्क लागू करे, सहेजे, और अगले फ़ाइल पर जाने से पहले `Watermarker` को बंद करे।
